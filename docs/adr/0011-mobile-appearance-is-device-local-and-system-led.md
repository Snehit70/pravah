# Mobile appearance is device-local and system-led

Pravah mobile defaults to the device's live appearance while retaining explicit
Warm light and Dark overrides. The preference stays device-local because ambient
light and system settings belong to the device, and legacy light-only
installations migrate once to System because their stored light value was not a
functional appearance choice.

Dark appearance uses stable aubergine-charcoal surfaces with warm text.
User-selected accents color interactive emphasis only; semantic task colors and
surface identity remain fixed. Every mobile surface and adjacent system chrome
must change together, meet WCAG AA, and never pass through a mixed-theme frame.

Kairo remains open when intentionally backgrounded, but a fresh launch mounts it
fully closed with no visible sheet chrome. Web appearance is outside this
decision.

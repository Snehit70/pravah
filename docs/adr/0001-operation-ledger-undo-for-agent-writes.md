# Operation ledger undo for agent writes

Pravah's CLI is an agent automation interface first, and its write surface is expected to reach phone feature parity before extending into agent-native workflows. Every agent write will create a first-class operation ledger record, and undoability will be a property of that operation rather than a separate caller convention. Single-step operations are the default, while grouped operations are supported for multi-command agent workflows and can be undone in reverse order when each recorded step has enough before-state. The operation ledger is a shared product safety primitive; the CLI may adopt it first, but phone undo flows should converge on the same backend model where practical. This keeps simple writes easy while making richer changes such as task updates, Goal deletion, Goal Link changes, and grouped agent actions reversible through a consistent `operations undo` contract.

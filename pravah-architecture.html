Pravah Architecture

Timeline-first task manager: React web, Expo mobile, Convex backend, Google sync, Kairo AI copilot, and MCP bridge.

Client Surface

  • • Web app: Vite + React 19 in src/
  • • Mobile app: Expo React Native in apps/mobile/
  • • Timeline, inbox, drag reorder, QuickAdd, settings
  • • Kairo client config stays local to the device

Backend + Data

  • • Convex queries, mutations, actions, and HTTP routes
  • • Better Auth sessions enforce per-user ownership
  • • Core tables: tasks, users, integrations, sync state
  • • MCP stdio bridge exposes task tools to agents

Integrations + Trust

  • • Google OAuth PKCE exchanges through Convex HTTP routes
  • • Calendar sync imports events into Pravah tasks
  • • Gmail candidates flow through a review queue
  • • User LLM API keys are called browser-side, not stored server-side
